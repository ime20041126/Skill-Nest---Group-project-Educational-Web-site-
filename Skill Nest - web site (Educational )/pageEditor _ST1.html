Volunteer Page

Technical Implementation

The Volunteer Page was built with:

  • HTML5 Structure: Semantic elements (<header>, <main>, <section>) for accessibility
  • CSS Flexbox: Responsive card layout for volunteer opportunities
  • JavaScript: Form popup functionality with openForm() and closeForm() functions
  • Form Validation: HTML5 validation attributes combined with JavaScript
  • Interactive Elements: Hover effects on cards using CSS transitions

Key Code Snippets

Volunteer Card Structure:

<div class="card">
    <img src="volunteer1.jpg" alt="Tutoring program">
    <h3>Online Tutoring</h3>
    <button onclick="openForm('Online Tutoring')">Apply Now</button>
</div>

Popup Form JavaScript:

function openForm(programName) {
    document.getElementById("program").value = programName;
    document.getElementById("application-form").style.display = "block";
}

Accessibility Features

  • Proper form labels with <label> elements
  • ARIA attributes for screen readers
  • Color contrast ratio of at least 4.5:1 for text
  • Keyboard navigable interface
  • Semantic HTML structure

Landing Screen

Technical Implementation

The Landing Screen includes:

  • Auto-Redirect: <meta http-equiv="refresh"> after 4 seconds
  • Fixed Header: Logo stays visible with CSS positioning
  • Background Image: Full-screen background with background-size: cover
  • Team Table: Responsive table with hover effects
  • Responsive Design: Adapts to mobile and desktop

Key Code Snippets

Auto-Redirect Meta Tag:

<meta http-equiv="refresh" content="4;url=homepage.html">

Background Styling:

body {
    background: url(bg-image.jpg);
    background-repeat: no-repeat;
    background-size: cover;
}

Accessibility Features

  • Proper table headers (<th>)
  • Alt text for logo image
  • Sufficient color contrast
  • Clear visual hierarchy
  • Warning about auto-redirect for screen readers

Challenges and Solutions

Volunteer Page Challenges

  • Form Validation: Initially had issues with complex validation rules. Solved by combining HTML5 validation with JavaScript for better user feedback.
  • Card Layout: Cards wouldn't align properly on mobile. Fixed with flexbox properties and media queries.

Content Page Challenges

  • Code Formatting: Code snippets didn't stand out. Added custom styling to <code> elements for better readability.
  • Content Overflow: Long module names caused layout issues. Implemented proper text wrapping and padding.

Landing Screen Challenges

  • Background Image: Initially didn't cover the whole screen. Fixed with background-size: cover.
  • Auto-Redirect: Needed to make it accessible. Added a visible countdown and ARIA alert.
